  - `Images/trains/*.bmp`：训练用的位图样本（文件名形如 `1058_2_131356675.bmp`）。
  - `Images/design/04模型训练操作.jpg`：Qt 工具界面参考图（中文）。
  - `.idea/*`：PyCharm 工程配置（项目 SDK 名称为 `PythonOcr`）。

## 你将要实现的“端到端”目标（两阶段 + 一个桌面工具）
1. **YOLOv13 定位模型**：训练“字符区域检测”（检测框）。
2. **YOLOv13 识别模型**：训练“字符识别”（可用分类或检测+类别做法，取决于数据标注方式）。
3. **Qt 桌面端训练工具**：界面参考 `Images/design/04模型训练操作.jpg`，所有提示中文。
4. **最终 OCR 推理**：先定位（检测）→ 再裁剪 ROI → 再识别（分类/检测），输出文本。

## 数据与命名约定（从仓库可观测信息推断）
- 样本位于：`Images/trains/`，格式：`.bmp`。
- 文件名呈 3 段下划线：`<id>_<k>_<stamp>.bmp`
  - 例如：`1_7_131830248.bmp`、`981_2_125409581.bmp`。
  - **注意：目前仓库里没有标注文件**（例如 YOLO 的 `*.txt` 或 COCO 的 `*.json`）。
- 建议在实现时：
  - 对“定位”使用 YOLO 检测标注：`labels/*.txt`（每行 `class x_center y_center w h`，归一化）。
  - 对“识别”如果走分类：需要 `class/` 结构或 `csv`/`txt` 标签映射；如果走检测+类别：同样是 YOLO 检测标注但类别为字符。

## 关键目录建议（落地代码时应新增/遵循）
- `src/`：训练/推理/数据集脚本（Python）。
- `qt_app/`：Qt 桌面程序（建议 PySide6，便于纯 Python 分发）。
- `datasets/`：
  - `datasets/det/`：定位数据（images/labels/train|val）。
  - `datasets/rec/`：识别数据（按选定方案组织）。
- `models/`：训练输出（权重、配置、日志）。
